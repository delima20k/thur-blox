# Admin Store

## Status

O repositorio nao possui autenticacao administrativa real. A aba Admin do frontend e apenas um contrato visual bloqueado.

## Painel alvo

O backend deve proteger o painel com autenticacao real e autorizacao por funcao. A lista de pedidos deve mostrar codigo, data, semente, quantidade, valor, desconto, pagamento, entrega, cliente e Roblox username.

## Acoes alvo

- Iniciar preparacao.
- Marcar cliente contatado.
- Agendar entrega.
- Iniciar entrega.
- Marcar como entregue.
- Marcar falha.
- Cancelar.
- Solicitar reembolso quando suportado.

Todas as acoes devem exigir confirmacao e gerar historico.
